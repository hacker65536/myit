モジュール1: 環境構築と設定

最初のモジュールでは、"検知と応答" をコントロールするための初期設定します。2つのCloudFormationテンプレートのうちの1つ目を実行して、これらのコントロールのいくつかの作成を自動化し、残りのものを手動で設定します。

AWS CloudFormationテンプレートのデプロイ

シナリオを開始して環境を設定するには、モジュール1用のCloudFormationテンプレートを実行する必要があります。

リージョン デプロイ
US West 2 (Oregon)

  1. 上記の [Deploy to AWS] ボタンをクリックします。 これにより、自動的にテンプレートが実行されます。 次へ ボタンをクリックします。

  2. [詳細の指定] セクションで、以下に示すように必要なパラメータを入力します。

    パラメーター 値
    Stack name ThreatDetectionWksp-Env-Setup
    Email Address 受信可能なメールアドレス (検知した際のアラートの通知先に使われます）

  3. パラメータを入力したら、[次へ] をクリックし、もう一度 [次へ] をクリックします（すべてをデフォルトのままにしておきます）

  4. 最後に、テンプレートよってIAMロールが作成されることを確認して [作成] をクリックします。

CloudFormation コンソールに戻ります。ページを更新して、作成を開始したスタックを確認することができます。次に進む前に、スタックのステータスが CREATE_COMPLETE になっていることを確認してください（下の図を参照）

  1. SNSからサブスクリプションを確認する電子メールが届きます。これを確認して、ワークショップ中にAWSサービスから電子メールアラートを受け取ることができます。
    アラートメールの送信元は <no-reply@sns.amazonaws.com> になります。受信できない場合は、ご利用のメール環境でフォルダ振り分けがされていたり、SPAM判定をされていないかご確認ください。

Amazon CloudWatchイベントルールと自動応答の設定

CloudFormationテンプレートは、アラートと応答の目的で3つの CloudWatch Events Ruleを作成しました。以下の手順で最終的なルールを作成します。この後、電子メール通知を受信し、応答のために適切な AWS Lambda 関数 をトリガーするための必要なすべてのルールが設定されます。

以下は、コンソールを使用してこのルールを作成する手順ですが、GuardDutyのドキュメントを確認することで、プログラムでルールを実行する方法についても詳しく知ることができます。

  1. CloudWatch コンソールを開きます。

  2. 左側のナビゲーションペインで、[イベント] の [ルール] をクリックします。

    現在実行中のルールは何かありますか？

  3. ルールの作成 をクリックします。

  4. イベントパターン の下の サービス別のイベントに一致するイベントパターンの構築 を選択し、ドロップダウンから カスタムイベントパターン を選択します。そして、以下のカスタムイベントパターンを貼り付けます。 

    {
  "source": [
    "aws.guardduty"
  ],
  "detail": {
    "type": [
      "UnauthorizedAccess:EC2/MaliciousIPCaller.Custom"
    ]
  }
}
    • ターゲットの追加 をクリック、Lambda関数 を選択して threat-detection-wksp-remediation-nacl を選択します。設定の詳細 をクリックします。

  5. ルールの詳細の設定 画面で、名前 と 説明 を入力します。

    • 名前: threat-detection-wksp-guardduty-finding-ec2-maliciousip
    • 説明: GuardDuty Finding: UnauthorizedAccess:EC2/MaliciousIPCaller.Custom

  6. ルールの作成 をクリックします。

  7. Optional: Lambda関数が何をしているのかを見てみましょう。Lambda コンソール を開きます。threat-detection-wksp-remediation-nacl という名前の関数をクリックします。

Amazon GuardDuty の有効化

次のステップでは、Amazon GuardDutyを有効にして、悪意のある、または許可されていない動作を継続的に監視します。

  1. Amazon GuardDuty コンソールに移動します

  2. [今すぐ始める] ボタンをクリックします。

  3. 次の画面で、[GuardDutyの有効化] ボタンをクリックします。

GuardDutyが有効になり、環境内の脅威発見のため CloudTrailログ、VPCフローログ、およびDNSクエリログを継続的に監視します。

Amazon Macie の有効化

機密データをS3に保存する予定なので、すぐにAmazon Macieを有効にしましょう。Macieは、不正アクセスや偶発的なデータ漏洩のリスクを検出した場合にデータアクセスアクティビティを継続的に監視し、アラートを生成するセキュリティサービスです。

  1. Amazon Macie コンソールに移動します。

  2. Get Started をクリックします。

  3. 有効にすると、Macieはサービスにリンクされたロールを作成します。ロールの権限を表示する場合は、View service role permissions をクリックします。

  4. Enable Macie をクリックします。

データの発見と分類のための Amazon Macie のセットアップ

Macieは機密データの自動検出と分類にも使用されます。S3バケットのデータ分類を統合するための設定します。

  1. Amazon Macie の左側のナビゲーションで Integrations をクリックします。

  2. AWS アカウントID を見つけて、SELECT をクリックします。

  3. ADD をクリックし、次の画面で「-data」で終わるS3バケットの横にあるチェックボックスをクリックします。ADD をクリックします

  4. Lオプションをデフォルトのままにして、Review をクリックします。

  5. 次の画面で、Start Classification をクリックします。

  6. 最後に Done をクリックします。Macie が有効になり、データの検索、分類、保護が開始されました。

AWS Security Hub の有効化

検知を管理するための設定が完了しました。AWS Security Hub を有効にする必要があります。これはAWS環境のセキュリティとコンプライアンスの包括的なビューを提供します。

  1. AWS Security Hub に移動します。

  2. Enable Security Hub をクリックします。

  3. Enable AWS Security Hub をクリックします。

AWS Security Hubが有効になり、これまでに有効にしたセキュリティサービスからの調査結果の収集と集約が開始されます。
